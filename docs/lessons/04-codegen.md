# Lesson 4 — Codegen (Rust → TypeScript)

## What you'll learn

The single most important mechanism in the project: how the Rust `SelfId` is turned
**automatically** into a TypeScript type, and how a "drift guard" makes sure the two can
never silently disagree. This is the idea from [Lesson 0](00-the-big-picture.md), made
concrete.

## The concept

The browser is written in TypeScript and needs to know the shape of `SelfId` so it can
work with it safely. The naive approach is to *hand-write* a TypeScript copy:

```ts
// DON'T do this — a hand-maintained duplicate that will drift
interface SelfId { id: string; name: string; fw_version: string; capabilities: string[]; }
```

The problem: when someone adds a field to the Rust `SelfId`, they'll forget to update
this copy, and now the board sends data the browser doesn't expect. Two sources of
truth = bugs.

Our approach: **generate** the TypeScript from the Rust. The pipeline is:

```
Rust SelfId  --(compile to WebAssembly)-->  contract.wasm
             --(wasm-bindgen reads it)----->  contract.d.ts (TypeScript types)
             --(we copy + add a banner)----->  web/src/contract.gen.ts
```

Three tools make this work:

- **`tsify`** (`tsify-next`) — a Rust library. When you put `#[derive(Tsify)]` on a
  struct, it teaches the Rust compiler how that struct should look in TypeScript.
- **`wasm-bindgen`** — the bridge between Rust-compiled-to-WebAssembly and JavaScript/
  TypeScript. It reads the compiled output and emits a `.d.ts` TypeScript declarations
  file.
- **WebAssembly (wasm)** — a portable compile target. We compile `contract` to wasm not
  to *run* it, but because that's the format `wasm-bindgen` understands.

You don't run these by hand. One command — `just gen` — does the whole pipeline.

## Walk the code

### The Rust side (from `contract`)

Recall these lines from [Lesson 2](02-the-contract.md), in `crates/contract/src/lib.rs`:

```rust
#[cfg_attr(feature = "codegen", derive(Tsify))]
#[cfg_attr(feature = "codegen", tsify(into_wasm_abi, from_wasm_abi))]
#[derive(Serialize, Deserialize, Debug, Clone, PartialEq)]
pub struct SelfId { ... }
```

The `Tsify` derive is only active under the `codegen` feature, so it adds zero weight to
the firmware or simulator builds. It only switches on when we're generating TypeScript.

### One subtlety: the exact `wasm-bindgen` version

In `crates/contract/Cargo.toml`:

```toml
wasm-bindgen = { version = "=0.2.100", optional = true }  # must equal devbox wasm-bindgen-cli version
```

The `=0.2.100` is an **exact** pin (note the `=`). The `wasm-bindgen` Rust *library* and
the `wasm-bindgen` command-line *tool* (installed by devbox, [Lesson 1](01-the-dev-environment.md))
must be the same version, or generation fails. The `=` stops Cargo from quietly drifting
to a newer patch.

### The `just gen` recipe (from `justfile`)

```bash
gen:
    #!/usr/bin/env bash
    set -euo pipefail
    out="$(mktemp -d)"
    cargo rustc -p contract --no-default-features --features codegen \
        --target wasm32-unknown-unknown --crate-type cdylib
    wasm-bindgen --target bundler --out-dir "$out" --out-name contract \
        target/wasm32-unknown-unknown/debug/contract.wasm
    mkdir -p web/src
    {
        echo "// @generated by \`just gen\` — do not edit"
        cat "$out/contract.d.ts"
    } > web/src/contract.gen.ts
    rm -rf "$out"
    echo "wrote web/src/contract.gen.ts"
```

Step by step:

1. `cargo rustc ... --crate-type cdylib --target wasm32-unknown-unknown` — compile the
   `contract` crate to WebAssembly, with the `codegen` feature on. The `--crate-type
   cdylib` is requested *here on the command line* rather than in `Cargo.toml`, so normal
   builds never produce wasm — only this command does.
2. `wasm-bindgen ... contract.wasm` — read the wasm and emit TypeScript declarations into
   a temp directory.
3. Prepend a `// @generated ... — do not edit` banner and write the result to
   `web/src/contract.gen.ts`.
4. Clean up the temp directory.

### The generated file: `web/src/contract.gen.ts`

```ts
// @generated by `just gen` — do not edit
/* tslint:disable */
/* eslint-disable */
/**
 * The molecule's self-description — the spine every later rung extends.
 */
export interface SelfId {
    id: string;
    name: string;
    fw_version: string;
    capabilities: string[];
}
```

Note: `id: String` in Rust became `id: string` in TypeScript; `Vec<String>` became
`string[]`. Even the doc comment carried over. **This file is committed to git** but you
never edit it by hand — the banner says so. The browser imports `SelfId` from here
([Lesson 5](05-the-web-workbench.md)).

### The drift guard: `just check-gen`

Committing a generated file raises a question: what if someone changes the Rust but
forgets to re-run `just gen`? Then the committed `.gen.ts` is stale. The guard catches
exactly that:

```bash
check-gen:
    #!/usr/bin/env bash
    set -euo pipefail
    just gen
    if ! git diff --exit-code -- web/src/contract.gen.ts; then
        echo "ERROR: web/src/contract.gen.ts is stale — run \`just gen\` and commit." >&2
        exit 1
    fi
    echo "contract.gen.ts is up to date"
```

It regenerates the file, then asks git: *did anything change?* If yes, the committed
version was out of date → fail the build with a clear message. If no change, you're in
sync. This runs as part of `just check` ([Lesson 7](07-the-spine.md)), so drift can
never sneak past the acceptance gate.

## Run it yourself

Generate the TypeScript type:

```bash
devbox run -- just gen
# wrote web/src/contract.gen.ts
```

Confirm the guard passes (nothing drifted):

```bash
devbox run -- just check-gen
# contract.gen.ts is up to date
```

### Try it yourself — watch the guard fire

```bash
# Hand-tamper the generated file (simulating drift):
printf '\n// tampered\n' >> web/src/contract.gen.ts

devbox run -- just check-gen; echo "exit=$?"
# Prints the "stale" error and exit=1 — the guard caught it.

# Restore the real generated file:
git checkout -- web/src/contract.gen.ts
devbox run -- just check-gen
# back to "up to date"
```

For a deeper experiment: add a field to `SelfId` in the Rust contract, run `just gen`,
and watch `web/src/contract.gen.ts` gain the field automatically — no hand-editing.
(Revert afterward unless you mean to keep it.)

## Recap

- The browser's `SelfId` type is **generated** from the Rust `SelfId`, never hand-written
  — that's how the two can't drift apart.
- The pipeline: compile `contract` to **WebAssembly** → **`wasm-bindgen`** emits TS
  declarations → write to **`web/src/contract.gen.ts`** with a "do not edit" banner.
- **`just gen`** runs the pipeline; **`just check-gen`** fails the build if the committed
  file is stale.
