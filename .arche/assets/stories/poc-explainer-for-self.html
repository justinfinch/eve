What the POC actually builds

A desk-scale, browser-driven replica of Adom's entire product loop — design, wire, measure, understand — built on Adom's own stack.

The short version: the POC builds a small modular board — a "molecule" — that announces what it can do on a bus, is driven entirely from a web browser, can measure and actuate a real circuit remotely, and has an AI layer that runs a test and explains the result in plain English — with a shareable link so anyone can reproduce it. Every piece is a recognizable slice of Adom's real product, not a generic Arduino demo. — POC: Mini-Molecule + Cloud Workbench

What it physically does

  • GPIOFlashes / toggles an LED on and off.
  • PWMDims that LED — or spins a small DC fan/motor at variable speed.
  • ADCReads a sensor (pot or thermistor) and streams the live value to the browser.
  • BOptional: software-wires a component into the measurement path, then measures it.
  • DStretch: actually turns stepper motors via Gcode typed in the browser.

The point isn't the blinking LED — it's that the whole loop runs end to end: browser → board → real circuit → live readout → AI explanation → shareable permalink. — POC page; demo parts in the hardware BOM.

From the block diagram to the breadboard

Wiring it up — the separate-parts bench build

This is the physical wiring for the separate-parts build (Pico 2 + standalone MCP2515 modules, kept apart from the all-in-one Waveshare board for learning value). The CAN side crosses a 5V/3.3V boundary, so the SPI bus runs through a level shifter. — wiring + gotchas mirror the build note in the hardware BOM.

3.3V power 5V power SPI signal MISO — the 5V→3.3V line CAN bus (differential)

MISO (SO on the module) is the one line the module drives back to the Pico — at 5V it can damage a 3.3V GPIO, so it must cross the shifter. The shifter handles the other SPI lines too (3.3V→5V), since the MCP2515's logic-high threshold is marginal at a bare 3.3V. Both CAN modules carry an onboard 120Ω terminator — correct for a 2-node bus.

Why it lands

The "so what"

The killer move is building on Adom's own MIT/Apache-licensed crates — postcard-rpc (messaging), mcp2515 (CAN), tsify (Rust→TS types), and the Embassy fork (firmware). That turns the project from "I made a demo" into "I used your stack to build a mini version of your product." — POC page citing Adom, Decoded.

The takeaway

You should now be able to sketch the path — browser → bridge → molecule → circuit — from memory, and say for each concept what it builds and which Adom subsystem it mirrors, and why A + B together prove the five pillars on Adom's own stack.
